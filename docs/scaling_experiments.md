# Scaling Experiments

This document is the detailed experiment note for the thesis project
**Comparative Evaluation of Quantum Encodings for Knowledge Graphs**. The main
user guide is [README.md](../README.md); this file focuses on scaling design,
output groups, append behavior, and interpretation.

The scaling experiments compare:

- basis encoding
- amplitude encoding
- phase encoding

The purpose is to study the KG-to-quantum workflow itself: RDF parsing,
deterministic ID mapping, state preparation, circuit construction, local
simulation, measurement, and resource metadata. The experiment is not a DBpedia
benchmark and does not claim quantum advantage.

## Experiment Groups

Outputs are separated into three groups:

| Group | Purpose | Output folder |
| --- | --- | --- |
| Synthetic | Controlled scalability analysis with fixed triples/entities/predicates. | `results/scaling/synthetic/` |
| Real | Real KG realism check using local professor-provided resources. | `results/scaling/real/` |
| Combined | Synthetic and real rows plotted together. | `results/scaling/combined/` |

## Synthetic Datasets

Synthetic datasets are deterministic Turtle files generated by:

```powershell
python scripts/generate_scaling_datasets.py
```

Current configurations:

| File | Triples | Entities | Predicates |
| --- | ---: | ---: | ---: |
| `data/scaling/synthetic_100.ttl` | 100 | 50 | 5 |
| `data/scaling/synthetic_1000.ttl` | 1,000 | 500 | 10 |
| `data/scaling/synthetic_5000.ttl` | 5,000 | 2,500 | 15 |
| `data/scaling/synthetic_10000.ttl` | 10,000 | 5,000 | 20 |

These files use triples of the form:

```turtle
@prefix ex: <http://example.org/> .

ex:e0 ex:p0 ex:e1 .
ex:e1 ex:p1 ex:e2 .
```

To add another synthetic size, edit `DATASET_CONFIGS` in
`scripts/generate_scaling_datasets.py`, then regenerate the files.

## Real KG Datasets

Real KG files are expected locally in:

```text
data/real_kgs/
```

Current default files:

```text
data/real_kgs/exampleV3.ttl
data/real_kgs/productsSmall.rdf
data/real_kgs/DecodedOntologies_V2.ttl
data/real_kgs/Aristotle.xml
```

These files come from the professor's QuantumEncoDeco resources. The scripts
prefer this local folder and do not download the files. If a file is missing or
cannot be parsed, the row is recorded with `status=parse_error` or `status=error` and the batch
continues.

Supported parser suffixes are `.ttl`, `.rdf`, `.xml`, and `.nt`. If you add a
new real KG, place it in `data/real_kgs/` and pass it with `--real-files`, for
example:

```powershell
python scripts/run_all_experiments.py --skip-synthetic --real-files my_graph.ttl --repetitions 1 --shots 256 --timeout-seconds 300
```

To include a file by default, add it to `REAL_KG_FILES` in
`scripts/run_scaling_experiments.py`.

## Incremental Runs

Use `--append` when you want to run more configurations later without replacing
the existing raw CSV.

Example:

```powershell
python scripts/run_scaling_experiments.py --sizes 100 1000 --repetitions 5
python scripts/run_scaling_experiments.py --sizes 5000 --encodings amplitude --repetitions 1 --append
python scripts/run_scaling_experiments.py --regenerate-plots-only
```

Append mode:

1. Loads the existing raw CSV.
2. Computes a run key.
3. Skips already-recorded matching configurations.
4. Runs new configurations.
5. Writes the merged raw CSV.
6. Recomputes the summary CSV.
7. Regenerates plots from all stored rows.

The run key includes:

```text
dataset_name
dataset_category
encoding
repetition
shots
max_basis_simulation_qubits
max_phase_diagonal_qubits
max_metric_qubits
timeout_seconds
weight_strategy
phase_marker_mode
phase_mark_predicate / phase_requested_predicate / phase_effective_predicate
phase_angle
decompose_reps
compute_decomposed_metrics
compute_transpiled_metrics
rdf_format
```

Use a separate output folder for experiments that should not touch the main
results:

```powershell
python scripts/run_scaling_experiments.py --sizes 5000 --repetitions 1 --output-dir results/scaling/test_5000
```

The all-in-one runner supports the same idea:

```powershell
python scripts/run_all_experiments.py --synthetic-sizes 5000 --encodings amplitude --repetitions 1 --append
python scripts/run_all_experiments.py --regenerate-plots-only
```

## Phase Predicate Selection

Phase encoding is meaningful only when the oracle marks at least one triple. The
synthetic datasets are built with predicates such as `http://example.org/p0`, so
synthetic phase runs default to `synthetic-default` and mark that predicate.
Real KGs usually do not contain the synthetic predicate, so real KG runs default
to `most-common-predicate`.

Available modes:

| Mode | Behavior |
| --- | --- |
| `synthetic-default` | Mark `http://example.org/p0`. |
| `first-predicate` | Mark the first predicate in the parsed deterministic triple list. |
| `most-common-predicate` | Mark the most frequently occurring predicate. |
| `none` | Mark no triples; explicit no-op/baseline. |
| `custom` | Mark `--phase-mark-predicate`; this argument is required. |

Phase-specific raw columns:

- `phase_marker_mode`
- `phase_requested_predicate`
- `phase_effective_predicate`
- `phase_marked_triples`
- `phase_total_triples`
- `phase_marked_fraction`
- `phase_warning`

If `phase_marked_triples = 0`, the row remains valid audit data, but it should
be interpreted as a no-op phase run. The CSV warning is included specifically so
real KG phase results are not accidentally treated as meaningful predicate
marking when no triples matched.

## Circuit Metric Levels

Qiskit may count high-level instructions such as `initialize(...)` and
`unitary(...)` as one operation in the original circuit. The experiment runner
therefore records three circuit metric levels:

| Metric family | Meaning |
| --- | --- |
| `logical_*` | Metrics on the original high-level circuit. Useful pipeline metadata, not hardware gate estimates. |
| `decomposed_*` | Metrics after `circuit.decompose(reps=...)`, when allowed by `--max-metric-qubits`. |
| `transpiled_*` | Metrics after Qiskit `transpile(..., basis_gates=["u", "cx"], optimization_level=0)`, when allowed by `--max-metric-qubits`. |

The legacy `circuit_depth`, `gate_count`, and `operation_counts` columns are
kept as aliases for the logical metrics. Use the logical/decomposed/transpiled
names in new analysis and thesis figures.

Metric extraction can itself be expensive. If decomposed or transpiled metrics
are not available, the raw row records `metric_status` as `metric_limit`,
`not_requested`, or `error`, with details in `metric_error`. Missing points in
decomposed/transpiled plots are therefore honest omissions, not zero values.

## Simulator Limits

Basis encoding represents:

```text
subject_bits || predicate_bits || object_bits
```

The basis qubit count therefore depends on the number of unique entities and
predicates. A 5,000-triple synthetic dataset can require 28 basis qubits; a
10,000-triple synthetic dataset can require 31 basis qubits. Dense statevector
memory grows as `2^n`, so those runs may be skipped with
`status=simulator_limit`.

Phase encoding currently uses a dense diagonal oracle. The option
`--max-phase-diagonal-qubits` controls the guardrail for this dense matrix path.

Amplitude encoding usually needs fewer qubits because it operates over padded
triple indices, but generic state preparation and simulation still have
measurable cost.

## Status Values

| Status | Meaning |
| --- | --- |
| `success` | Full run completed. |
| `simulator_limit` | A configured dense-simulation guardrail was exceeded. |
| `timeout` | The run exceeded `--timeout-seconds`. |
| `parse_error` | RDF parsing failed for the dataset. |
| `error` | Missing file, memory error, invalid input, custom phase configuration error, or another exception. |
| `skipped` | Not normally written by current scripts; duplicate append runs are skipped in terminal output. Legacy/manual skipped rows are preserved if present. |

These statuses are part of the result. They show where the local workflow is
practical and where it is not.

## Parameter Notes

The current simulator guardrail parameters are:

- `--max-basis-simulation-qubits`, default `22`
- `--max-phase-diagonal-qubits`, default `10`
- `--max-metric-qubits`, default `14`

Circuit metric controls:

- `--decompose-reps`, default `1`
- `--compute-decomposed-metrics` / `--no-compute-decomposed-metrics`
- `--compute-transpiled-metrics` / `--no-compute-transpiled-metrics`

Phase marker controls:

- `--phase-marker-mode`
- `--phase-mark-predicate`
- `--phase-angle`

There is no backend-selection parameter. Scaling experiments use Qiskit's
`Statevector` object and write `Qiskit Statevector` in the `backend` column.

For the complete command-line table, see the README.
